# הסתברות זכייה — כלי מקומי (משחק 1)

כלי Monte-Carlo שמחשב לכל מתמודד את **הסיכוי לזכות בקופה** לפי המצב הנוכחי,
הברקט הנעול שלו, והמשחקים שנותרו. משתמש ישירות במנוע הניקוד של האתר
(`computeUserPoints`), אז הוא עקבי 100% עם החישוב האמיתי.

## מצב לייב (מומלץ) — רענון מושך תוצאות אמת

דאבל-קליק על **`serve-live.bat`** → נפתח שרת מקומי ודפדפן ב‑`http://localhost:5173`.
כל **רענון** של הדף (או כפתור "🔄 רענן עכשיו") מושך את **תוצאות האמת** של מונדיאל 2026
ממקור ציבורי אמין (openfootball), ומריץ מחדש את הסימולציה **רק כשמשהו השתנה**.

```bash
npx tsx server.ts          # ידני; אפשר SIMS=3000 PORT=5173
```

- מקור התוצאות: `https://raw.githubusercontent.com/openfootball/worldcup.json` — ללא מפתח, מתעדכן אחרי כל משחק.
- כרגע נמשכים אוטומטית משחקי **בתים**. תוצאות **נוקאאוט** מוסיפים ידנית ב‑`played-results.ts` (הן גוברות על הלייב).

## 📈 היסטוריית הסתברויות — איך זה השתנה לאורך הטורניר

כפתור **"📈 היסטוריית הסתברויות"** בדף הראשי (או `http://localhost:5173/history`) מציג
את התפתחות סיכוי הזכייה של כל מתמודד **מלפני המשחק הראשון ועד עכשיו** — נקודה לכל משחק.

- השחזור **דטרמיניסטי**: הכלי בונה מחדש את העקומה מ‑0 משחקים, ומוסיף משחק־משחק לפי
  **הסדר הכרונולוגי האמיתי** (תאריך/שעה מ‑openfootball). לכן אין צורך "לזכור" מה הוערך
  בהתחלה — תמיד אפשר לחזור ולראות את כל המסלול, והוא **מתארך אוטומטית** עם כל משחק חדש.
- גרף קווי (Chart.js): ציר X = 0 (לפני הטורניר) ואז מספר המשחק. כברירת מחדל מוצגים
  6 המובילים; לחיצה על שם בלגנדה מוסיפה/מסתירה מתמודד. ריחוף מציג איזה משחק נוסף בכל נקודה.
- מתחת לגרף: **טבלה מלאה** (מתמודד × משחק) עם חיצים ▲/▼ לשינוי מול הנקודה הקודמת.
- כל בנייה נשמרת גם ל‑**`history.json`** (ארטיפקט קבוע לעיון/גיבוי).

## מצב סטטי (קובץ HTML חד-פעמי)

דאבל-קליק על **`run-winprob.bat`**, או:

```bash
npx tsx winprob.ts 50000   # [מספר ריצות] [seed]
```

יוצר `winprob.html`. משתמש ב‑`played-results.ts` (עריכה ידנית), ולא דורש אינטרנט.

> דרוש Node.js מותקן. `tsx` יורד אוטומטית דרך `npx` (לא צריך `npm install`).

## עדכון ידני (למצב סטטי / נוקאאוט)

1. פתח את **`played-results.ts`** והוסף תוצאות (`A1..L6` לבתים, `'73'..'104'` לנוקאאוט).
2. הרץ שוב — ההסתברויות מתעדכנות. ככל שמתווספות תוצאות, ההערכות מתחדדות.

## מה רואים בפלט

הכלי יוצר **`winprob.html`** — דוח בטבלה מסודרת (עברית, RTL) שנפתח אוטומטית בדפדפן
בסיום הרצה דרך ה‑bat. בטבלה: דירוג, סיכוי זכייה (עם בר), טופ-3, נק' צפויות,
מיקום נוכחי, האלופה שכל אחד בחר, ו**עמודת הסבר קצר** למה האחוז כזה
(למשל: "לרעתו: צ׳כיה — כבר הפסידה").

בנוסף, בקונסול:
- **Current standings** — הדירוג הנוכחי (נקודות משחקים בלבד). שימושי לאימות.
- **WIN PROBABILITY** — לכל מתמודד: `win%`, `top3%`, `avgPts`, `nowRank`.
- **Turkey impact** — השוואה: עולם אמיתי (טורקיה הפסידה) מול תרחיש-נגד
  (המשחק שלה לא שוחק). Δ שלילי = נפגעת מההפסד של טורקיה, חיובי = הרווחת.

## איך לבדוק חשיפה לקבוצה אחרת (לא רק טורקיה)

ב‑`winprob.ts`, בלוק ה‑counterfactual מוחק את `D2` (פתיחת טורקיה).
כדי לבדוק קבוצה/משחק אחר, שנה את מזהה המשחק שנמחק, ועדכן את `turkeyDepth`
לקבוצה הרצויה.

## הנחות והגבלות

- **מודל גולים**: Poisson עם `λ = 1.3 × att × def` מתוך `src/pages/results/teamStrength.ts`.
  מגרש ניטרלי (אין יתרון בית). אפשר לכוונן את `BASE` ואת דירוגי הקבוצות.
- **נוקאאוט בתיקו**: מוכרע לפי הסתברות יחסית לחוזק (`λ_home/(λ_home+λ_away)`).
- **מלך השערים לא נכלל** (סימטרי וזניח בשלב זה) — זו הערכת זכייה **יחסית**, לא תחזית מדויקת.
- שובר-שוויון בעלייה משלב הבתים נפתר לפי לוגיקת האתר (`standings.ts` + מטריצת ההקצאה).

## מה נכלל בחישוב (זהה לכל המתמודדים, ללא עדיפות לאף אחד)

- **נקודות עלייה בכל שלב**: בית 4, R32 7, R16 8, רבע 12, חצי 16, מקום 3 → 20, אלופה 25.
  מי שלקח נבחרת רחוק גוזר את הנקודות רק כשהיא באמת מגיעה לשם בסימולציה.
- **ניקוד משחקים** (צליפה/כיוון) בבית ובנוקאאוט לפי הברקט המדויק.
- **נקודות מיקום** בבית.
- **מלך שערים** — 3 נק'/גול + 10 בונוס לזוכה (מודל ב‑`golden-boot.ts`).
- **המצב הנוכחי** של כל נבחרת נלקח בחשבון דרך 8 התוצאות המקובעות (כולל הפרש שערים),
  שמשנות את סיכויי העלייה שלה בכל ריצה.

## קבצים רלוונטיים

- `serve-live.bat` — **מצב לייב** (שרת + דפדפן, רענון = תוצאות אמת).
- `server.ts` — השרת המקומי (כולל `/history`).
- `live-results.ts` — משיכת תוצאות אמת + סדר כרונולוגי, ומיפוי לנבחרות/משחקים.
- `history.ts` — שחזור עקומת ההסתברויות לאורך הטורניר + גרף/טבלה.
- `history.json` — הסטוריה שמורה (נוצר אוטומטית).
- `sim-core.ts` — מנוע הסימולציה + הדירוג + ה‑HTML (משותף).
- `winprob.ts` — מצב סטטי (יוצר `winprob.html`).
- `played-results.ts` — תוצאות ידניות / נוקאאוט (גוברות על הלייב).
- `golden-boot.ts` — קצבי הכובשים (ניתן לעריכה).
- `run-winprob.bat` — הרצת המצב הסטטי בקליק.
